## bootstrap两种文件的区别
   - **.min.js .min.css** 这类文件是压缩后的文件，可以直接用于生产环境

   - **.js .css** 这个类文件是没有经过压缩的文件，主要用于开发环境方便 debug 问题

   ---

## bootstrap html 标准模板
   **可以用如下的标准方式引入 bootstrap 到 html 中**
   ```html
   <!DOCTYPE html>
   <html lang="zh-CN">
   
   <head>
       <meta charset="utf-8">
       <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">
       <meta http-equiv="x-ua-compatible" content="ie=edge">
       <!-- bootstrap css-->
       <link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/bootstrap/4.4.1/css/   bootstrap.min.css"
           integrity="sha384-Vkoo8x4CGsO3+Hhxv8T/Q5PaXtkKtu6ug5TOeNV6gBiFeWPGFN9MuhOf23Q9Ifjh"    crossorigin="anonymous">
   
       <title>Hello, bootstrap!</title>
   </head>
   
   <body>
   
   
       <!-- boostrap 依赖于 jquery 和 popper 所以要先引入他们-->
       <script src="https://code.jquery.com/jquery-3.4.1.slim.min.js"
           integrity="sha384-J6qa4849blE2+poT4WnyKhv5vZF5SrPo0iEjwBvKU7imGFAV0wwj1yYfoRSJoZ+n"
           crossorigin="anonymous"></script>
       <script src="https://cdn.jsdelivr.net/npm/popper.js@1.16.0/dist/umd/popper.min.js"
           integrity="sha384-Q6E9RHvbIyZFJoft+2mJbHaEWldlvI9IOYy5n3zV9zzTtmI3UksdQRVvoxMfooAo"
           crossorigin="anonymous"></script>
       <script src="https://stackpath.bootstrapcdn.com/bootstrap/4.4.1/js/bootstrap.min.js"    integrity="sha384-wfSDF2E50Y2D1uU
       </body>
   </html>
   ``` 
   **注释**
   ```html
   <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">
   <!--1、UI布局的宽度与设备的宽度一致 2、缩放大小为原始大小 3、-->
   ```
